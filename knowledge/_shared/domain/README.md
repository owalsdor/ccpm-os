# Shared Domain Knowledge

*What things are*, across two or more projects — vocabulary, stakeholders, conventions, product context that is portfolio-wide.

Organize as small, focused files (one topic per file). Suggested shape:

```
_shared/domain/
├── README.md              # this file
├── glossary.md            # cross-project vocabulary
├── stakeholders.md        # who's who, org chart, approvers
├── naming-conventions.md  # how things are named
└── {topic}.md             # add as needed
```

Entries graduate here from a project's `domain.md` once the same lesson appears in 2+ projects (see [`../../INDEX.md`](../../INDEX.md) → Graduation path).
