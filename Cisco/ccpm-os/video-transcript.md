## 1. Hook / The Problem (0:00 - 0:30)

**NARRATOR:**

How much time do you spend every week building the same PM artifacts from scratch? PRDs, competitive battlecards, roadmaps, meeting summaries, sales enablement kits. You know the format. You know what good looks like. But every time, you're starting from a blank page — re-inventing the structure, copy-pasting from old docs, and spending hours on formatting instead of thinking.

What if you had an operating system that already knew the playbook?

## 2. What is CCPM-OS (0:30 - 1:15)

**NARRATOR:**

CCPM-OS is a markdown-first operating system for product managers, built to run inside Cursor. It gives you 40 slash commands and 70 reusable skills — covering everything from strategy and execution to market research, go-to-market, and personal productivity.

`[SCREEN: Show the repository folder structure — commands/ and skills/ directories expanding to reveal categories: execution, go-to-market, market-research, product-strategy, toolkit]`

The system has two layers. **Commands** are the workflows — you invoke them with a slash, give them your input, and they walk you through a structured process. **Skills** are the playbooks underneath — the detailed templates, frameworks, and best practices that commands draw from.

`[SCREEN: Simple diagram — a "Command" box (e.g., /write-prd) with an arrow pointing to a "Skill" box (e.g., create-prd), with the output flowing to a "PRD Document" box]`

You don't need to manage any of this. Clone the repo, open it in Cursor, type a slash — and you're running.

---

## 3. How It Works — Examples (1:15 - 2:45)

**NARRATOR:**

Let me show you what this looks like in practice.

### Example 1: Writing a PRD

Say you have a feature idea. You open Cursor chat and type:

`[SCREEN: Cursor chat panel. User types: /write-prd SSO support for enterprise customers]`

The command asks a few targeted questions — who's the user, what's the success metric, any constraints? Then it generates a full Product Requirements Document: executive summary, background, objectives with measurable success metrics, target users, prioritized user stories in P0/P1/P2 tiers, solution overview, open questions, and a phased timeline.

`[SCREEN: Scroll through a generated PRD — show the section headings, a success metrics table, and the P0 user stories table]`

What used to take an afternoon now takes minutes. And the structure is consistent every time.

### Example 2: Competitive Battlecard

Now say your sales team keeps running into a competitor. You type:

`[SCREEN: User types: /battlecard Our CRM vs Salesforce for mid-market teams]`

The command researches the competitor, then produces a one-page battlecard: where you win, where you lose, a feature comparison matrix, objection handling with proof points, landmine questions to plant, and conversation starters for displacement deals.

`[SCREEN: Show the battlecard output — highlight the "Quick Summary" section with "We win when / We lose when", the comparison table, and the objection handling section]`

Your sales reps get something they can actually use on a call — not a 20-page document nobody reads.

### Example 3: Weekly Brain Dump

This one is personal. Sunday evening, you paste everything on your mind — work tasks, personal commitments, half-formed ideas — into:

`[SCREEN: User types: /weekly-braindump and pastes a messy, unstructured wall of text]`

The command sorts every item into categories, decides what to do, schedule, delegate, defer, or delete — then builds a day-by-day weekly plan with themed days, deep work blocks protected, a delegation list with owners and due dates, and a calendar-ready event table you can paste directly into your calendar tool.

`[SCREEN: Show the structured output — the weekly plan with themed day headers, a delegation table, and the calendar event table]`

From chaos to a clean week in two minutes.

## 4. Benefits (2:45 - 3:30)

**NARRATOR:**

Four things make this system different.

**Speed.** Artifacts that took hours now take minutes. The structure is already there — you focus on the content and decisions, not the formatting.

`[SCREEN: Side-by-side — "Before: 3 hours" vs "After: 5 minutes" with a PRD icon]`

**Consistency.** Every PRD, every battlecard, every roadmap follows the same proven structure. Your team speaks the same language, and stakeholders always know where to find what they need.

**Composability.** Commands chain together naturally. Write a PRD, then run `/write-stories` to generate user stories from it, then `/test-scenarios` to create QA scenarios. Each output feeds the next.

`[SCREEN: Flow diagram — /write-prd arrow to /write-stories arrow to /test-scenarios, each producing a document]`

**Customizable.** The whole system is just markdown files in a repo. Want a new workflow? Add a command. Want to change the PRD template? Edit the skill. Your team's best practices become reusable infrastructure, not tribal knowledge locked in someone's head.

## 5. Getting Started (3:30 - 4:00)

**NARRATOR:**

Getting started takes about 30 seconds.

Clone the repository. Open the folder in Cursor. That's it.

`[SCREEN: Terminal showing "git clone" command, then Cursor opening the folder]`

Cursor automatically picks up the commands and skills. Type `/` in the chat panel, and you'll see all 40 commands organized by category. Pick one, paste your input, and go.

One thing to know: by default, the system plans first and waits for your approval before making changes. It asks at most two clarifying questions, then assumes sensible defaults. You stay in control.

`[SCREEN: Show the Cursor slash command menu with categories visible]`
